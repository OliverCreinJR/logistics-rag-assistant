# Logistics RAG Assistant

Веб-ассистент, отвечающий на вопросы перевозчиков и складских сотрудников по корпоративным регламентам. Семантический поиск + генерация ответа на основе найденного контекста (RAG-архитектура).

Pet-проект для портфолио. Является эволюцией более раннего корпоративного Telegram-бота: вместо жёстко прописанного меню — понимание смысла свободно сформулированных вопросов.

## Архитектура

```
            ┌──────────────────┐
   вопрос ─►│  Streamlit UI    │
            └────────┬─────────┘
                     │
                     ▼
            ┌──────────────────┐
            │  query embedding │  multilingual-e5-small
            └────────┬─────────┘
                     │
                     ▼
            ┌──────────────────┐
            │ vector search    │  ChromaDB (cosine, top-3)
            │   in ChromaDB    │
            └────────┬─────────┘
                     │
                     ▼
            ┌──────────────────┐
            │  prompt builder  │  system + user prompt
            └────────┬─────────┘
                     │
                     ▼
            ┌──────────────────┐
            │  Groq (Llama 3.3 │  temperature=0.1, защита от галлюцинаций
            └────────┬─────────┘
                     │
                     ▼
              ответ + источники
```

**Стек:**
- Эмбеддинги: `intfloat/multilingual-e5-small` (sentence-transformers, locally hosted)
- Векторное хранилище: ChromaDB (persistent, локально)
- Чанкование: `langchain-text-splitters` (RecursiveCharacterTextSplitter, 500 символов / 100 overlap)
- Генерация: Groq API (Llama 3.3 70B Versatile)
- UI: Streamlit
- Деплой: Streamlit Community Cloud (free tier)

## Запуск локально

### 1. Окружение

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Ключ Groq

1. Зарегистрируйтесь на [console.groq.com](https://console.groq.com/keys).
2. Создайте API-ключ.
3. Скопируйте `.env.example` в `.env` и впишите ключ:

```bash
cp .env.example .env
# отредактируйте .env: GROQ_API_KEY=<ваш_ключ>
```

> Без ключа приложение работает в режиме «только retrieval» — показывает найденные фрагменты регламентов без сгенерированного ответа.

### 3. Индексация регламентов

```bash
python ingest.py
```

Скрипт прочитает все `.md` из `docs/`, нарежет на чанки, посчитает эмбеддинги и сохранит в `chroma_db/`. Запускается один раз при первом запуске или при изменении документов.

### 4. Запуск UI

```bash
streamlit run app.py
```

Откройте http://localhost:8501.

### 5. Оценка качества retrieval

```bash
python eval.py
```

На выходе — таблица Hit Rate @ 1 / @ 3 / @ 5 и детальный отчёт `eval_results.md`.

## Деплой на Streamlit Cloud

1. Запушьте проект в публичный GitHub-репозиторий.
2. Зайдите на [share.streamlit.io](https://share.streamlit.io), нажмите **New app**.
3. Укажите репозиторий, ветку, файл `app.py`.
4. В **Advanced settings → Secrets** добавьте:

```toml
GROQ_API_KEY = "ваш_ключ"
```

5. Deploy. При первом запуске приложение автоматически соберёт индекс из `docs/` (~1–2 минуты).

> **Замечание про SQLite:** Streamlit Cloud по умолчанию идёт со старой версией `sqlite3`, которую ChromaDB не поддерживает. В `requirements.txt` подключён `pysqlite3-binary`, а в `ingest.py`/`rag.py` есть автоматический свап модулей. На локальной машине этот код no-op.

## Дизайн-решения

**Почему e5-small, а не e5-base?**
Для базы из 8 документов разница в точности минимальна (1–3 п.п. на типичных Hit Rate @ 3), а размер модели в 4 раза меньше (118 МБ vs 478 МБ). Это критично для бесплатного тарифа Streamlit Cloud (1 ГБ RAM). При переезде на платный хостинг — поменять одну строку в `config.py`.

**Почему Groq + Llama 3.3 70B?**
Globally accessible API (работает без VPN в любой точке мира), бесплатный тариф с щедрыми лимитами, очень высокая скорость генерации (~500 токенов/сек). Модель Llama 3.3 70B уверенно работает с русским языком. Groq-совместим с OpenAI SDK — смена провайдера сводится к замене `base_url` и `api_key`.

**Почему `temperature=0.1` и явный запрет галлюцинаций в системном промпте?**
Юзкейс — корпоративные процедуры. Цена выдуманного срока подачи претензии или артикула — реальные деньги. Лучше сказать «не знаю» и перенаправить на старшего смены, чем уверенно ответить неправильно. Системный промпт явно требует ссылаться только на контекст.

**Почему чанки 500 / overlap 100?**
Регламенты структурированы: разделы по 1–3 абзаца, средний абзац ~300–600 символов. Чанк 500 захватывает связный смысловой блок целиком; overlap 100 страхует на случай, когда ответ лежит на стыке абзацев. Сплиттер уважает заголовки `##` и `###` — режет в осмысленных местах.

**Почему ChromaDB, а не FAISS / Pinecone?**
Чистая локальная инсталляция без серверов и платных сервисов. На таком объёме данных (≈100 чанков) производительность не важна, а простота разворота — важна.

## Возможные улучшения

- Reranker поверх retrieval (`BAAI/bge-reranker-v2-m3`) — обычно даёт +5–15 п.п. к Hit Rate @ 1
- Подмешивание keyword-поиска (BM25) для запросов с редкими терминами (ТТН, ДОПОГ, ТОРГ-2)
- Структурированный вывод: на вопросы вида «составь акт о повреждении для машины X от даты Y» — заполнение шаблона через function calling
- Multi-hop RAG для вопросов, требующих информации из нескольких регламентов одновременно
- Пользовательский фидбек (👍 / 👎) с логированием для последующего fine-tuning подбора промпта
